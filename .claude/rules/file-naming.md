# File Naming

Use these conventions so files sort chronologically and self-describe:

- **Journal entries:** `YYYY-MM-DD-DAY.md` (e.g., `2026-04-21-TUE.md`)
- **All other generated files:** `YYYY-MM-DD_topic_type.md` (e.g., `2026-04-21_user-auth_prd.md`)

The day-of-week suffix on journals uses the three-letter uppercase abbreviation (`MON`, `TUE`, `WED`, `THU`, `FRI`, `SAT`, `SUN`).

For non-markdown outputs (e.g., style guides in JSON), adapt the pattern: `YYYY-MM-DD_topic_type.json` or — for named artifacts — `<company>-style-guide.json`.
